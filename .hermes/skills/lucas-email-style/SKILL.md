# Lucas Hanson Email Style

Use this skill whenever drafting emails for Lucas. Update this skill if Lucas provides feedback on structure or style.

## Core Principles

- **Direct, no fluff:** Jump straight to purpose. No "hope you're doing well" filler
- **Concise but complete:** Provide context efficiently, not more than needed
- **Polite but firm:** Respectful without being overly formal
- **Bilingual:** Same structure in Danish and English, just different language

## Structure Patterns

### Basic Structure

```
Greeting,
[1-2 paragraphs: context + ask/problem]
Closing,
[Name]
```

### Greetings

- **Danish:** "Hej [Navn]," (casual) or "Hej [Company]," (business)
- **English:** "Hi [Name]," or "Hey [Name]," (slightly more casual)

Always use recipient's name when known.

### Body Paragraphs

**Direct questions:**
- Danish: "Jeg ville derfor høre om..." / "Jeg ville lige høre om der er noget jeg aktivt skal gøre..."
- English: "I thus wanted to hear if..." / "I would like to ask if..."

**Stating situation:**
- Danish: "Jeg står i den situation at..."
- English: "I am in the situation where..."

**Thanking:**
- Danish: "Mange tak på forhånd," / "Tak for beskeden,"
- English: "Thanks in advance," / "Thanks for message,"

### Closings

- **Danish:** "Venlig hilsen," (always)
- **English:** "Best regards," (business) / "Regards," (slightly less formal)

Never vary these - Lucas uses these consistently.

### Signature

Include signature for formal/business contexts (support, job applications, business inquiries):

**Danish:**
```
Lucas Hanson
E-mail: [email]
Mobil: (+45) 20673709
Webside: lucashanson.dk
```

**English:**
```
Lucas Hanson
E-mail: [email]
Mobile: (+45) 20673709
Website: lucashanson.dk
```

Skip signature for casual/personal emails (friends, quick responses).

## Length Guidelines

- **Ultra-short:** Single link or confirmation (e.g., "https://photos.app.goo.gl/..." or just "Det er gjort.")
- **Short:** 1-2 sentences for simple asks (account deletion, basic info)
- **Medium:** 2-3 paragraphs for situations requiring context (job inquiries, support issues)
- **Long:** Only when complex explanation needed (LinkedIn access issue with full background)

## Context-Specific Variations

### Job Applications (Danish)
```
Hej [Name],

Tak for at række ud.
[Why interested - 1 paragraph]
[Experience/fit - 1 paragraph]
[Availability + CV mention]

Lucas Hanson

```

### Support Tickets (English)
```

[Brief problem statement]

```

### Business/Professional (Danish)
```
Hej [Name/Company],

[Polite acknowledgment if replying]

Lucas Hanson

```

### Personal/Casual (Danish/English)
```
Hej/Hi [Name],

[Direct message - 1-2 lines max]

Venlig hilsen/Best regards,
Lucas
```

## Language Rules

- **Danish emails:** Use Danish greetings and closings, Danish sentence structure
- **English emails:** Use English greetings and closings, English sentence structure
- **Mixed context:** If recipient might not speak Danish, ask: "I also speak Danish. I just wasn't sure if you do as well."
- **Form vs address:** "Hej [Company]," is fine in Danish when writing to businesses

## Common Phrases

**Asking for info:**
- Danish: "Jeg ville høre om der er noget jeg aktivt skal gøre..."
- English: "I would like to ask if there is something I need to actively do..."

**Stating intent:**
- Danish: "Jeg ville derfor gerne høre om..."
- English: "I thus wanted to hear if..."

**Thanking after meeting:**
- Danish: "Jeg ville lige skrive og takke for i går. Det var fedt..."
- English: "I just wanted to write and thank you for yesterday. It was..."

**Problem solving:**
- Danish: "Jeg har over meget af dagen prøvet at... Jeg kan dog ikke komme igennem..."
- English: "I have tried to... all day. However, I cannot get through..."

## What to Avoid

- ❌ "Håber du har det godt" / "Hope you're doing well"
- ❌ Excessive pleasantries or filler
- ❌ Over-formal language in either language
- ❌ Rambling or unnecessary detail
- ❌ Dashes or em dashes in email body unless user explicitly wants them
- ❌ Variation in closings - always "Venlig hilsen," / "Best regards,"
- ❌ First-person pronouns when not needed (keep it action-focused)

## Scheduling Replies

When drafting a response about meetings, interviews, calls, or other time-sensitive invites, check the user's calendar or stated availability first if the environment allows it. Base the reply on actual free/busy time, not guesswork. If calendar access is unavailable, say that plainly and ask the user for their availability before drafting.

## Examples

### Danish - Job Inquiry
```
Hej Signe,

Tak for lige at række ud til min moder.
Shape lyder som et ekstremt spændende sted. Jeg så på jeres hjemmeside at I både er interesseret i studerende indenfor backend såvel som iOS og Android udvikling. Gennem mit studie har vi haft mest fokus på backend men har også haft fag til Android udvikling som jeg også har været meget glad for.
Så jeg har en stor interesse indenfor for alt det i søger men mest erfaring indenfor backend og Android udvikling.
Hvis det passer, vil jeg også meget gerne snakke og jeg har tid hele dagen i dag, dog indtil 17.

Ser frem til at høre tilbage,

Lucas Hanson
```

### English - Support Request
```
Hi Nightpay,

I previously created my Nightpay account on my Android (and I am pretty sure I chose a Google account login option). Now, when I try to log in to my account on my new iPhone, it asks for phone number of that account, but I did not never get to set that up.
Also, I have tried resetting the password on your website (though I never received an email), changing the email on the app I have on my Android (but it is not possible), and also looked at the article you have on login (but nothing worked).
So basically, right now, to get access to my membership, I have to bring my old phone with me. It would be nice if you could maybe add my phone number or something to my account so I can just log in to it on my iPhone.

Best regards,

Lucas Hanson
```

### Danish - Business
```
Hej Colosseum klinikken,

Jeg har over meget af dagen i dag prøvet at afbestille min tandlæge tid på fredag. Jeg kan dog ikke komme igennem, da i ifølge mobilens svarer har travlt.
Jeg håber min tid kan afmeldes gennem denne mail. Jeg prøver dog også at ringe igen i morgen.

Venlig hilsen,

Lucas Hanson
```

### Ultra-short (Danish)
```
Hej Michael,

Det er gjort. Den skal bare lige godkendes.

Venlig hilsen,

Lucas Hanson
```
